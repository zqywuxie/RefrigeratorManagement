```text
请对现有“实验室冰箱管理系统（Laboratory Freezer Management System）”进行一次完整的 UI/UX 与冰箱空间管理 Redesign。

注意：
- 系统主体功能已经存在
- 本次重点不是重构系统架构
- 而是重新设计：
  - 冰箱空间布局
  - 样本定位逻辑
  - 抽屉与盒子交互
  - 样本孔位管理
  - 真实冰箱映射
  - 大规模样本操作体验
  - 样本分组逻辑
  - 二维可视化界面
  - Excel样本导入与拖拽映射

系统用于：
- 生物医学实验室
- 孕产相关样本库
- 外周血 / 脐血 / 足底血 等样本管理

整体风格要求：
- Benchling + Notion + LIMS 风格
- 极简科技风
- 深色模式兼容
- 强空间层级感
- 支持高密度数据展示
- 支持快速搜索与快速定位

====================================================
一、冰箱结构设计
====================================================

系统包含两个冰箱。

====================================
冰箱1（Drawer Freezer）
====================================

------------------------------------
上层区域（开放式存储）
------------------------------------

- 共 2 行开放式储物层
- 不使用固定格子
- 不强制精确空间坐标
- 更偏向“物品管理”
- 类似开放货架

支持存放：
- 试剂
- 样本盒
- 耗材
- 临时物品
- 离心管架

核心要求：

如果用户选择：
- “试剂”
- “样本盒”
- “样本”

则其内部管理逻辑需要与下层抽屉中的 Box 完全一致。

即：

上层存放的“样本盒”也支持：
- 盒子管理
- 孔位管理
- 样本录入
- 拖拽
- 二维孔位映射
- 批量导入

因此：

上层不仅仅是简单物品列表，
而是：
- 普通物品模式
- Box模式
两种模式共存。

界面要求：

- 卡片式自由布局
- 支持拖拽排序
- 支持标签分类
- 支持搜索
- 支持最近活跃高亮
- 支持颜色分类

====================================================
下层区域（抽屉式）
====================================================

下层采用抽屉式结构。

------------------------------------
第一层抽屉区
------------------------------------

- 共 6 个抽屉
- 固定布局：
  - 2 行 × 3 列

------------------------------------
第二层抽屉区
------------------------------------

- 共 15 个抽屉
- 固定布局：
  - 5 行 × 3 列

所有抽屉支持：

- 抽屉动画
- hover预览
- 点击展开
- 热力图占用状态
- 空位统计
- 最近访问时间

每个抽屉内：

- 可放多个 Box
- Box 支持自由拖拽排序
- Box 支持二维布局展示
- Box 支持缩略图模式
- Box 支持批量移动

要求：

用户能够通过拖拽：
- 调整 Box 顺序
- 更换 Box 所属抽屉
- 批量移动多个 Box

界面类似：
- 文件管理器
- Kanban
- 仓储货架系统

====================================================
二、Box（样本盒）Redesign
====================================================

每个 Box 支持两种模式：

------------------------------------
模式1：孔位模式（核心）
------------------------------------

用于：
- 冻存管
- 单细胞样本
- 血液样本
- DNA/RNA样本

支持：
- 10×10
- 9×9
- 8×12
- 自定义孔位布局

要求：

- 使用二维孔位可视化
- 每个孔位支持点击
- 支持颜色标记
- 支持 hover 样本信息
- 支持批量导入
- 支持 Excel 导入

新增字段：

Box 支持可选字段：
- 数据路径（Data Path）

用于：
- 原始数据目录
- 测序结果目录
- NAS路径
- 本地服务器路径
- 云端数据路径

例如：
- /data/project/sample_001/
- \\NAS\lab\freezer\box_A01\
- s3://bucket/project/

要求：
- 支持点击复制
- 支持路径跳转预留
- 支持多个路径
- 支持路径标签

====================================================
三、样本设计（核心）
====================================================

一个真实样本可能需要：
- 分为多个冻存试剂管
- 占据多个孔位

例如：

“张三 外周血”
可能分为：
- A01
- A02
- A03

三个试剂管。

因此系统需要支持：

====================================
一个样本 → 多个试剂管 → 多个孔位
====================================

要求：

用户新增样本时：

- 可以点击多个孔位
- 将多个孔位绑定为同一个样本
- 自动生成多个 Tube
- Tube 属于同一个 Sample

例如：

点击：
- A01
- A02
- A03

系统自动生成：

Sample：
- 张三 / PBMC / 中孕期

Tubes：
- Tube1 → A01
- Tube2 → A02
- Tube3 → A03

要求：

- 同一样本的多个孔位使用统一颜色
- 支持 grouped highlight
- 支持 grouped hover
- 支持 grouped search
- 支持 grouped move
- 支持 grouped delete

视觉要求：

- 同一样本形成“关联态”
- 可使用：
  - 连线
  - glow
  - 相同边框
  - 同色系

====================================================
四、样本字段设计（核心）
====================================================

样本字段固定为：

必填字段：
- 姓名
- 编号

可选字段：
- 样本来源
- 样本类型
- 采集阶段
- 采集时间
- 标签
- 备注

系统自动字段：
- 上传者（默认当前登录用户）

要求：

====================================
批量编辑支持（重点）
====================================

以下字段支持：
“多选批量修改”

包括：

- 样本来源
- 样本类型
- 采集阶段
- 采集时间

即：

用户可一次选择多个样本，
统一修改上述字段。

例如：
- 批量设置为：
  - 中孕期
  - 外周血
  - 门诊

要求：
- 支持批量覆盖
- 支持批量追加
- 支持撤销
- 支持操作历史

====================================================
五、Excel 导入 + 拖拽映射设计（重点）
====================================================

系统需要支持：

用户上传 Excel 后：

====================================
右侧：
====================================

展示：
“待分配样本列表”

每一行代表：
一个样本。

要求：

- 支持搜索
- 支持筛选
- 支持颜色标签
- 支持 hover 查看详情
- 支持拖拽

====================================================
Excel字段识别（重点）
====================================================

系统需要自动识别 Excel 字段。

重点识别：

- 编号
- 姓名
- 住院号
- 年龄

支持：
- 自动字段匹配
- 用户手动重新映射字段

例如：

Excel列：
- patient_name
- sample_id
- inpatient_no

用户可以手动选择映射为：
- 姓名
- 编号
- 住院号

要求：

- 上传后进入“字段映射界面”
- 支持下拉选择字段对应关系
- 支持保存字段模板
- 支持记忆用户习惯
- 支持预览前10行数据

====================================================
左侧：
====================================================

展示：
二维冰箱 + Box 孔位布局。

用户操作流程：

1. 上传 Excel
2. 自动识别字段
3. 用户确认字段映射
4. 右侧生成待分配样本列表
5. 左侧显示冰箱二维结构
6. 用户拖拽样本到孔位
7. 支持一次点击多个孔位
8. 一个样本可绑定多个孔位
9. 系统自动生成多个 Tube

例如：

拖拽：
“张三 外周血”

到：
- A01
- A02
- A03

系统自动生成：
- Tube1
- Tube2
- Tube3

并绑定到：
同一个 Sample。
```
